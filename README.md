# Электронный журнал

Веб-приложение для работы с учебным журналом. Студент видит расписание, оценки и лабораторные работы. Преподаватель ведёт занятия, выставляет оценки, отмечает посещаемость и проверяет загруженные решения.

## Возможности

- вход и регистрация пользователей;
- роли `Студент` и `Преподаватель`;
- расписание занятий;
- журнал оценок, пропусков и опозданий;
- оценки в журнале и лабораторных работах по шкале от 1 до 10;
- добавление занятий преподавателем;
- программа предмета с лабораторными работами, дедлайнами, материалами и ТЗ;
- загрузка решений лабораторных работ студентом;
- проверка решений преподавателем;
- комментарии к сданным работам;
- локальное хранение файлов;
- хранение данных в SQL Server.

## Технологии

- React + TypeScript + Vite;
- Node.js + Express;
- SQL Server Express;
- JWT для авторизации;
- локальная папка `uploads/submissions` для файлов.

## Что нужно установить

- Node.js 20 или новее;
- SQL Server Express;
- утилиту `sqlcmd`.

По умолчанию проект подключается к локальному серверу `.\SQLEXPRESS` через Windows-аутентификацию.

## Настройка

1. Установите зависимости:

```bash
npm install
```

2. Создайте файл `.env` по примеру `.env.example`:

```env
DB_SERVER=.\SQLEXPRESS
DB_DATABASE=Gradebook
DB_TRUSTED_CONNECTION=true
DB_TRUST_SERVER_CERTIFICATE=true
JWT_SECRET=gradebook-local-dev-secret
PORT=3000
```

3. Создайте базу данных и начальные записи:

```bash
npm run db:init
```

Скрипт создаёт базу `Gradebook`, таблицы, связи и стартовые данные. При запуске сервер также проверяет наличие таблиц и создаёт их, если база пустая.

## Запуск

```bash
npm run dev
```

После запуска откройте:

```text
http://localhost:3000
```

## Стартовые пользователи

В базе заранее создан один преподаватель и один студент.

| Роль | Логин | Пароль |
| --- | --- | --- |
| Преподаватель | `teacher` | `teacher123` |
| Студент | `student` | `student123` |

## Как войти преподавателем

1. Откройте страницу приложения.
2. Оставьте вкладку `Вход`.
3. Введите логин `teacher`.
4. Введите пароль `teacher123`.
5. Нажмите `Войти`.

После входа преподаватель может открыть журнал, добавить занятие, выставить оценку, отметить отсутствие или опоздание, настроить программу предмета и проверить лабораторные работы.

## Как войти студентом

1. Откройте страницу приложения.
2. Оставьте вкладку `Вход`.
3. Введите логин `student`.
4. Введите пароль `student123`.
5. Нажмите `Войти`.

После входа студент видит расписание, журнал оценок и список лабораторных работ. В разделе лабораторных можно выбрать работу, прикрепить файл решения и написать комментарий преподавателю.

## Регистрация

На экране входа откройте вкладку `Регистрация`.

Для студента нужно заполнить:

- ФИО;
- электронную почту;
- роль `Студент`;
- группу;
- логин;
- пароль.

Для преподавателя нужно заполнить:

- ФИО;
- электронную почту;
- роль `Преподаватель`;
- логин;
- пароль.

После успешной регистрации пользователь сразу входит в систему. Новый преподаватель появится в системе без закреплённых предметов. Предметы можно добавить в базу данных или расширить проект отдельным экраном администрирования.

## База данных

SQL-скрипт находится в:

```text
src/server/init.sql
```

Основные таблицы:

- `Users`;
- `Subjects`;
- `Lessons`;
- `Grades`;
- `SyllabusItems`;
- `LabSubmissions`;
- `SubmissionPartners`;
- `FeedbackComments`.

Файлы лабораторных работ сохраняются локально в `uploads/submissions`, а в SQL Server хранится путь к файлу и данные о сдаче.
